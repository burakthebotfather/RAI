# tg-rating-bot

Telegram-бот для оценки качества работы водителей доставки.

## Алгоритм работы

1. Водитель ставит `+` в **thread A** нужного чата
2. Если с момента последнего `+` прошло ≥ 8 часов — бот:
   - Отправляет уведомление в **thread B** с текущим рейтингом
   - Рассылает опрос (кнопки 1–5⭐) двум администраторам в личные сообщения
3. Администратор нажимает оценку → бот благодарит его
4. Оценка сразу учитывается в рейтинге водителя
5. Сообщения с кнопками и подтверждением удаляются через 24 часа

## Формула рейтинга

```
рейтинг = сумма всех оценок / количество оценок
```

Начальный рейтинг (без оценок): **5.0**

```
├── database.py     # работа с SQLite
├── requirements.txt
├── Procfile
└── .gitignore
```

## Деплой на Railway

1. Создай проект на [railway.app](https://railway.app) и подключи репозиторий
2. Добавь переменную окружения:
   - `BOT_TOKEN` — токен от [@BotFather](https://t.me/BotFather)
3. Railway подхватит `Procfile` и запустит бота как worker

> ⚠️ SQLite-база (`ratings.db`) хранится на диске контейнера.  
> При рестарте Railway данные сохраняются, но при полном пересоздании тома — нет.  
> Для продакшена рекомендуется подключить Railway Postgres и адаптировать `database.py`.

## Добавление нового чата

В `config.py` добавь запись в словарь `CHATS`:

```python
-1009999999999: {
    "name": "Название организации",
    "thread_a": 10,   # thread_id, где водитель ставит +
    "thread_b": 20,   # thread_id, куда бот пишет уведомление
    "admin1": 123456789,
    "admin2": 6792310907,
},
```
